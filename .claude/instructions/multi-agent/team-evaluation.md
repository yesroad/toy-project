# Agent Teams 규칙 준수 평가

> 팀 리드가 각 팀원의 작업 결과를 평가하는 템플릿
> 10개 항목 100점 만점

---

## 평가 시점

- 모든 팀원이 작업 완료 보고를 마친 후
- 팀 리드가 통합 검증 전에 수행

---

## 평가 항목 (100점 만점)

### 1. CLAUDE.md 규칙 참조 (10점)

| 점수 | 기준 |
|------|------|
| 10 | 프로젝트 규칙 파일을 읽고 구조/컨벤션을 정확히 따름 |
| 5 | 일부 규칙만 참조하거나 부분적으로 따름 |
| 0 | 규칙을 무시하고 자의적으로 구현 |

### 2. 코딩 표준 준수 (15점)

| 점수 | 기준 |
|------|------|
| 15 | import 순서, 네이밍 컨벤션, 타입 정의 등 모두 준수 |
| 10 | 대부분 준수하나 1-2개 미준수 |
| 5 | 절반 정도만 준수 |
| 0 | 표준을 무시 |

### 3. React 컨벤션 준수 (15점)

| 점수 | 기준 |
|------|------|
| 15 | 컨테이너/뷰 패턴, 훅 사용, 상태 관리 경계 모두 준수 |
| 10 | 대부분 준수하나 소소한 위반 |
| 5 | 패턴을 부분적으로만 따름 |
| 0 | 컨벤션을 무시 |

### 4. 디자인 구현 품질 (10점)

| 점수 | 기준 |
|------|------|
| 10 | 디자인을 참조하여 정확히 구현 (디자인 작업 시) |
| 5 | 디자인을 참조했으나 일부 차이 |
| 0 | 디자인 미참조 또는 디자인 작업 아님 (N/A) |

> 디자인 작업이 아닌 경우 이 항목은 N/A로 표시하고 나머지 항목에서 비례 배분

### 5. 스타일 컴포넌트 패턴 (10점)

| 점수 | 기준 |
|------|------|
| 10 | 스타일 컴포넌트를 별도 파일로 분리하고 named import 사용 |
| 5 | 스타일은 분리했으나 import 패턴이 프로젝트 관행과 다름 |
| 0 | inline style 남용 또는 스타일 파일 미분리 |

### 6. SendMessage 보고 품질 (10점)

| 점수 | 기준 |
|------|------|
| 10 | 변경 파일, 요약, 검증 결과, 주의사항을 모두 포함한 구조화된 보고 |
| 5 | 기본 정보만 포함 (변경 파일 목록 정도) |
| 0 | 보고 없음 또는 "완료" 한마디만 |

### 7. Done 프로세스 수행 (10점)

| 점수 | 기준 |
|------|------|
| 10 | teammate-done-process.md의 전체 단계 수행 완료 |
| 5 | 일부 단계만 수행 |
| 0 | done 프로세스 미수행 |

### 8. 범위 밖 변경 금지 (10점)

| 점수 | 기준 |
|------|------|
| 10 | 요청받은 범위 내에서만 변경 |
| 5 | 범위 밖 변경이 있으나 합리적 (import 수정 등) |
| 0 | 불필요한 범위 밖 변경 (리팩토링, 디자인 값 변경 등) |

### 9. 커밋 컨벤션 준수 (5점)

| 점수 | 기준 |
|------|------|
| 5 | `type: 메시지` 형식 완전 준수, Co-Authored-By 없음 |
| 3 | 형식은 맞으나 메시지가 부정확 |
| 0 | 형식 미준수 또는 Co-Authored-By 포함 |

### 10. BDD/테스트 시나리오 (5점)

| 점수 | 기준 |
|------|------|
| 5 | 정책 키워드가 있으면 테스트 작성, 없으면 테스트 불필요 판단 명시 |
| 3 | 테스트 필요성 인지했으나 미작성 |
| 0 | 정책 변경에 대한 테스트 고려 없음 |

---

## 등급 기준

| 등급 | 점수 | 설명 |
|------|------|------|
| **A** | 90+ | 우수 - 모든 규칙을 정확히 따름 |
| **B** | 70-89 | 양호 - 대부분 준수하나 소소한 미흡 |
| **C** | 50-69 | 보통 - 개선 필요한 항목 존재 |
| **D** | 50 미만 | 미흡 - 규칙 재교육 필요 |

## 주의사항

- 평가는 팀 리드만 수행 (팀원은 자기 평가 불가)
- 디자인 구현 항목이 N/A인 경우 90점 만점으로 환산
- 평가 결과는 다음 Agent Teams 세션에 반영 (프롬프트 개선)
