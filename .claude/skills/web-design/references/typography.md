# 2026 웹/앱 디자인 타이포그래피 가이드

---

## 1. 자간 (Letter Spacing / Tracking)

### 헤딩의 자간 축소 경향

2025~2026년 부상한 '청키(Chunky)' 타입(낮은 x높이의 뚱뚱한 서체)은 헤딩에서 자간을 좁히는 방향으로 사용합니다:

- **대형 헤드라인:** -0.01em ~ -0.02em (letter-spacing: -1 ~ -2%)
- **효과:** 응집력 있는 인상 형성

### 본문 자간은 기본값 유지

- 기본 자간(0) 또는 극히 미세한 조정만 권장
- 자간을 과도하게 늘리면 단어 인식 속도 저하

### WCAG 2.1 접근성 기준 (AA 준수 필수)

W3C WCAG 1.4.12(Text Spacing) 기준:

- **자간:** 사용자가 폰트 크기의 0.12배 이상으로 늘렸을 때도 콘텐츠 손실 없음
  - 예: 16px 폰트라면 자간이 약 2px까지 넓어져도 레이아웃 유지
- **단어 간격:** 폰트 크기의 0.16배 이상으로 설정해도 콘텐츠 손상 없음

## 2. 줄 간격 (Line Height / Leading)

줄 간격은 가독성에 가장 직접적인 영향을 주는 요소입니다. 2026년 트렌드는 줄 간격을 넓게 가져가는 방향입니다.

### 권장 줄 간격 값

| 용도                | 권장 line-height |
| ------------------- | ---------------- |
| 본문 텍스트 (일반)  | 1.4 ~ 1.7        |
| 본문 텍스트 (최적)  | 1.5 ~ 1.6        |
| 대형 헤딩 (H1)      | 1.1 ~ 1.3        |
| 중간 헤딩 (H2~H3)   | 1.2 ~ 1.4        |
| 짧은 캡션/UI 레이블 | 1.2 ~ 1.4        |

### 단위: 없는 상대값 사용 (필수)

```css
/* ✅ 권장 */
line-height: 1.5;

/* ❌ 비권장 */
line-height: 24px;
```

**이유:** 폰트 크기가 바뀌어도 자동으로 비례합니다. web.dev를 포함한 모든 주요 레퍼런스가 강하게 권장합니다.

### WCAG 접근성 기준

- **최소 줄 간격:** 폰트 크기의 1.5배 이상 (WCAG 1.4.12)
- **난독증 사용자:** 1.5 이상 특히 효과적
- **시각 약자:** 1.5~2.0 권장

### 문단 간격

- **최소 기준:** 폰트 크기의 2배 이상
- **중요:** 줄 내부 간격보다 문단 사이 간격이 반드시 더 크게 설정되어야 구분이 명확

## 3. 폰트 크기와 타이포그래피 스케일

### 브레이크포인트별 권장 폰트 크기

| 구분     | 모바일 (~599px) | 태블릿 (600~991px) | 데스크톱 (992~1439px) | 울트라와이드 (1440px+) |
| -------- | --------------- | ------------------ | --------------------- | ---------------------- |
| **본문** | 16px            | 16~18px            | 16~18px               | 17~18px                |
| **H1**   | 28~32px         | 32~40px            | 40~52px               | 44~56px                |
| **H2**   | 22~26px         | 24~32px            | 28~40px               | 34~44px                |
| **H3**   | 18~20px         | 18~22px            | 20~26px               | 24~28px                |

### 핵심 원칙: 본문 최소 16px

어떤 디바이스에서도 본문 텍스트는 16px 미만으로 내려가서는 안 됩니다. 이는 2026년 거의 모든 디자인 시스템이 명문화하고 있는 필수 원칙입니다.

### 유체 타이포그래피 (Fluid Typography)

CSS clamp()를 활용한 반응형 폰트 크기가 표준화되고 있습니다:

```css
h1 {
  font-size: clamp(2rem, 5vw, 3rem);
}
```

**특징:**

- 최솟값과 최댓값 사이에서 뷰포트에 따라 자연스럽게 스케일링
- 수많은 미디어 쿼리 없이 깔끔한 구현

**주의:** vw 단위 단독 사용은 사용자의 텍스트 크기 조절을 막으므로 금지합니다.

### 단위: rem 기반 상대값 사용

```css
/* ✅ 권장 */
font-size: 1.5rem;
padding: 2rem;

/* ❌ 비권장 */
font-size: 24px;
padding: 32px;
```

**이유:** rem을 사용하면 사용자의 브라우저 기본 글꼴 설정을 존중하고 접근성을 높입니다. 간격 토큰도 가능한 한 rem 단위로 통일하는 것이 권장 추세입니다.

## 4. 정렬 (Alignment)

### 좌측 정렬이 표준

본문 텍스트에는 **왼쪽 정렬(Left-align)**이 2026년 표준입니다:

- 읽기 방향(LTR 기준)에서 각 줄의 시작점이 일관됨
- 눈의 이동이 자연스러움
- WCAG도 우선 권장하며, 특히 난독증·저시력 사용자에게 가장 우호적

### 중앙/우측 정렬은 제한적 사용

- **중앙 정렬:** 짧은 캡션, 영웅(Hero) 섹션의 헤딩, 특정 강조 목적에만 사용
- **문제점:** 긴 본문에 적용 시 각 줄의 시작점이 들쑥날쑥해 읽기 흐름 방해

### 양쪽 정렬(Justified) 주의

정렬 방식 중 가장 논란이 많은 양쪽 정렬의 문제점:

- 하이픈(-)과 간격 조절이 완벽하지 않으면 시각적 '강(River)' 현상 발생
- WCAG는 접근성이 중요한 콘텐츠에서 사용 비권장
- **2026년 트렌드:** 대부분의 웹/앱에서 양쪽 정렬 지양

### 인터랙션 기반 동적 정렬

사용자 상호작용에 따라 요소가 미세하게 정렬 위치를 조정하거나 재배치되는 패턴이 증가하고 있습니다. 특히 모바일 앱의 카드/피드 UI에서 두드러집니다.

## 5. 콘텐츠 너비와 가독성 최적 선폭

### 최적 한 줄 글자 수

| 기준             | 권장 글자 수        |
| ---------------- | ------------------- |
| 일반 본문        | 50~75자 (영문 기준) |
| WCAG 권고 최대   | 80자 이하           |
| 다단 레이아웃    | 40~50자             |
| CJK(한중일) 언어 | 40자 이하           |

### CSS 구현 권장 방법

```css
article {
  max-width: 66ch;
}
```

**장점:** ch 단위는 해당 폰트의 '0' 글자 너비를 기준으로 하기 때문에 폰트 크기가 변해도 글자 수 기준이 유지됩니다.

### 최대 컨테이너 너비

| 디바이스      | 최대 콘텐츠 너비                      |
| ------------- | ------------------------------------- |
| 데스크톱      | 1140~1280px                           |
| 울트라와이드  | 1140~1280px 유지, 양 옆 여백으로 처리 |
| 블로그/아티클 | 640~760px                             |
| 랜딩 페이지   | 960~1140px                            |

**주의:** 울트라와이드 화면에서 텍스트를 전체 너비로 늘리는 것은 2026년 공통적으로 금기시되는 실수입니다.

---

## 6. 요약

2026년 타이포그래피의 핵심은 **일관성, 접근성, 유연성**입니다:

- 상대값(rem, %)을 통한 확장 가능한 설계
- WCAG 기준을 넘어선 의도적 여유(여백, 줄간격)
- 사용자의 환경 변화에 자동 대응하는 유체 타이포그래피
