# 2025–2026 폰트 추천

## 1. 전체 흐름 요약

2020년대 초반을 지배하던 산세리프 중심의 클린 미니멀 타이포그래피는 2023년을 기점으로 전환기를 맞이했다. 산세리프 폰트의 과포화로 차별화가 어려워지고, 팬데믹 이후 따뜻하고 인간적인 감성에 대한 수요가 높아지면서 세리프 폰트가 빠르게 귀환했다. 2025~2026년은 세리프와 산세리프가 공존하며 대비를 이루는 혼합의 시대다.

| 시기      | 2020~2022       | 2023~2024           | 2025~2026          |
| --------- | --------------- | ------------------- | ------------------ |
| 흐름      | 산세리프 전성기 | 세리프 귀환         | 혼합 + 가변의 시대 |
| 스타일    | Clean / Flat    | Elegant / Nostalgic | Bold + Editorial   |
| 대표 폰트 | Inter / Roboto  | Fraunces / Lora     | Variable Font 확산 |

## 2. 5대 타이포그래피 트렌드

**트렌드 1 — 에디토리얼 세리프의 귀환**
잡지·신문에서 볼 법한 두꺼운 세리프 헤드라인이 웹·앱 디자인으로 넘어왔다. 굵은 획과 가는 획의 대비(stroke contrast)가 크고 섬세한 세리프 폰트가 우아함과 고급스러운 인상을 준다. 패션, 뷰티, 저널리즘, 럭셔리, 금융, 인테리어 업종에서 특히 두드러진다.

**트렌드 2 — 타이포그래피가 비주얼이 된다**
이미지 없이 뷰포트를 꽉 채우는 초대형 헤드라인 텍스트만으로 첫인상을 완성하는 방식이 확산됐다. 폰트 자체가 브랜드 비주얼의 역할을 하며, `clamp()`를 활용한 반응형 유체 타이포그래피(Fluid Typography)가 표준 구현 방식으로 자리 잡았다.

**트렌드 3 — 가변 폰트(Variable Font) 확산**
하나의 폰트 파일로 굵기(weight), 너비(width), 기울기(slant)를 연속적으로 조절할 수 있는 가변 폰트가 빠르게 보급됐다. 파일 수와 용량이 줄어 로딩 속도가 개선되고, CSS 트랜지션으로 굵기 애니메이션도 가능해졌다. 구글 폰트의 주요 폰트 대부분이 가변 폰트로 전환되고 있다.

**트렌드 4 — 대비를 활용한 혼합 페어링**
같은 계열끼리 매칭하던 기존 방식에서 벗어나, 개성 있는 디스플레이 세리프 헤드라인과 깔끔한 산세리프 본문을 조합하는 대비 페어링이 주류가 됐다. 차이가 클수록 시각적 위계가 명확해지고 기억에 남는 인상을 준다.

**트렌드 5 — 모노스페이스 폰트의 일반화**
개발 환경에서만 쓰이던 모노스페이스 폰트가 일반 서비스 UI의 포인트 요소로 쓰이기 시작했다. 가격, 숫자 데이터, 날짜, 코드 스니펫 표기에 활용하여 테크·크리에이티브 감성을 연출한다.

## 8. 핵심 요약

2026년 폰트 트렌드의 본질은 **"단정함보다 개성, 균일함보다 대비"** 다. 산세리프 일색이던 웹 디자인에 에디토리얼 세리프가 귀환하면서 두 계열이 대비를 이루는 조합이 주류를 형성하고 있다. 가변 폰트의 확산으로 하나의 파일로 다양한 표현이 가능해진 것도 중요한 변화다.

한국어 서비스 기준으로는 **Pretendard를 본문 기본값으로 깔고, 헤드라인에만 트렌디한 포인트 폰트를 얹는 방식**이 가장 실용적이고 현재 가장 많이 보이는 패턴이다. 영문 헤드라인 폰트는 한글과 함께 쓸 때 반드시 CSS에서 언어별로 분리해 지정해야 한글이 깨지지 않는다.

## 9. Next.js next/font 구현 방법

### Google Fonts 폰트 (next/font/google)

Google Fonts에 있는 폰트는 `next/font/google`로 최적화 로드 가능.

```ts
// app/layout.tsx
import { Inter, Fraunces, Space_Grotesk, Noto_Serif_KR } from 'next/font/google'

const inter = Inter({
  subsets: ['latin'],
  variable: '--font-inter',
  display: 'swap',
})

const fraunces = Fraunces({
  subsets: ['latin'],
  variable: '--font-fraunces',
  axes: ['opsz', 'WONK'], // 가변 폰트 axes 활성화
  display: 'swap',
})

const notoSerifKR = Noto_Serif_KR({
  subsets: ['latin'],
  weight: ['400', '700', '900'],
  variable: '--font-noto-serif-kr',
  display: 'swap',
})

export default function RootLayout({ children }) {
  return (
    <html lang="ko" className={`${inter.variable} ${fraunces.variable} ${notoSerifKR.variable}`}>
      <body>{children}</body>
    </html>
  )
}
```

```ts
// tailwind.config.ts
fontFamily: {
  sans: ['var(--font-inter)'],
  display: ['var(--font-fraunces)'],
  'serif-kr': ['var(--font-noto-serif-kr)'],
}
```

### Pretendard (Google Fonts 미지원 → CDN 방식)

Pretendard는 Google Fonts에 없어서 `next/font/google`로 불러올 수 없다. CDN을 사용하거나 로컬 파일을 `next/font/local`로 등록한다.

**방법 1 — CDN (빠르게 시작할 때)**

```ts
// app/layout.tsx <head>에 추가 또는 next/head 사용
// CDN: https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/variable/pretendardvariable.css
```

```tsx
// app/layout.tsx
export default function RootLayout({ children }) {
  return (
    <html lang="ko">
      <head>
        <link
          rel="stylesheet"
          href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable-dynamic-subset.css"
        />
      </head>
      <body style={{ fontFamily: "'Pretendard Variable', Pretendard, sans-serif" }}>
        {children}
      </body>
    </html>
  )
}
```

**방법 2 — next/font/local (권장: 외부 의존성 없음)**

```bash
# public/fonts/ 에 파일 배치
# https://github.com/orioncactus/pretendard/releases 에서 다운로드
# PretendardVariable.woff2 사용 권장
```

```ts
// app/layout.tsx
import localFont from 'next/font/local'

const pretendard = localFont({
  src: '../public/fonts/PretendardVariable.woff2',
  variable: '--font-pretendard',
  display: 'swap',
  weight: '45 920', // 가변 폰트 weight range
})

export default function RootLayout({ children }) {
  return (
    <html lang="ko" className={pretendard.variable}>
      <body>{children}</body>
    </html>
  )
}
```

```ts
// tailwind.config.ts
fontFamily: {
  sans: ['var(--font-pretendard)', 'sans-serif'],
}
```

### 한글+영문 혼용 시 CSS 분리

영문 헤드라인 폰트(예: Fraunces)를 한글과 함께 쓸 때, 한글에 영문 폰트가 적용되지 않도록 반드시 `unicode-range`로 분리하거나 Tailwind에서 폰트 계층을 아래처럼 지정한다.

```ts
// tailwind.config.ts — 영문 헤드라인 + 한글 폰트 혼용 조합 예시
fontFamily: {
  // 영문은 Fraunces, 한글은 Noto Serif KR로 fallback
  'display': ['var(--font-fraunces)', 'var(--font-noto-serif-kr)', 'serif'],
  // 본문: 영문 Inter, 한글 Pretendard로 fallback
  'sans': ['var(--font-inter)', 'var(--font-pretendard)', 'sans-serif'],
}
```
