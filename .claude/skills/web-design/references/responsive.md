# 2026 웹/앱 디자인 반응형 브레이크포인트 가이드

---

## 1. 핵심 원칙: 모바일 퍼스트 (Mobile First)

2025~2026년 표준은 **모바일 퍼스트**입니다. CSS를 작성할 때:

- 가장 작은 화면을 기본으로 설정
- `min-width` 미디어 쿼리로 위로 쌓아올리기
- Tailwind, Bootstrap, Material Design 모두 동일한 방향

---

## 2. 주요 프레임워크별 브레이크포인트 비교

### Tailwind CSS

| 접두사        | min-width      | 대상 기기                   |
| ------------- | -------------- | --------------------------- |
| (없음 / 기본) | 0px            | 모바일 (모든 크기)          |
| `sm`          | 640px (40rem)  | 대형 스마트폰 / 소형 태블릿 |
| `md`          | 768px (48rem)  | 태블릿                      |
| `lg`          | 1024px (64rem) | 소형 데스크톱 / 가로 태블릿 |
| `xl`          | 1280px (80rem) | 대형 데스크톱               |
| `2xl`         | 1536px (96rem) | 울트라와이드                |

### Bootstrap 5

| 클래스      | 범위        | 대상 기기     |
| ----------- | ----------- | ------------- |
| `xs` (기본) | ~576px 미만 | 스마트폰 세로 |
| `sm`        | 576px~      | 스마트폰 가로 |
| `md`        | 768px~      | 태블릿        |
| `lg`        | 992px~      | 소형 데스크톱 |
| `xl`        | 1200px~     | 대형 데스크톱 |
| `xxl`       | 1400px~     | 초대형 화면   |

### Material Design 3

| 클래스        | 너비        | 대표 기기             |
| ------------- | ----------- | --------------------- |
| `Compact`     | ~599dp      | 세로 폰               |
| `Medium`      | 600~839dp   | 세로 태블릿, 폴더블   |
| `Expanded`    | 840~1199dp  | 가로 태블릿, 데스크톱 |
| `Large`       | 1200~1599dp | 데스크톱              |
| `Extra-Large` | 1600dp~     | 울트라와이드          |

## 3. 2025년 실제 화면 해상도 TOP 6 (BrowserStack 기준)

1. **1920×1080** — 데스크톱 FHD (가장 많음)
2. **360×800** — 안드로이드 스마트폰
3. **390×844** — iPhone 14/15 기준
4. **393×873** — Pixel 계열
5. **1366×768** — 노트북/저가 모니터
6. **1536×864** — 노트북 HiDPI

## 5. 2025~2026년 주목할 트렌드 변화

### 콘텐츠 기반 브레이크포인트

특정 기기 폭에 맞추는 것이 아니라 **레이아웃이 실제로 깨지는 지점**을 찾아 그곳에 브레이크포인트를 설정합니다:

- ❌ "iPad 너비에 맞춰라"
- ✅ "내 콘텐츠가 어색해지는 지점에 맞춰라"

### 컨테이너 쿼리 (Container Queries)

브레이크포인트의 패러다임을 변화시키고 있습니다:

**기존 방식 (미디어 쿼리)**

- 뷰포트(화면 전체) 너비를 기준으로 스타일 변경

**새로운 방식 (컨테이너 쿼리)**

- 부모 요소의 크기를 기준으로 스타일 변경
- Tailwind v4에 `@container`, `@sm`, `@md` 등 정식 내장
- 컴포넌트가 배치되는 위치에 따라 자동으로 레이아웃 변경
- 재사용성 대폭 증가

### 브레이크포인트 수 유지

**권장:** 5단계 이하 유지

- 너무 많으면 유지보수 복잡
- 대부분의 프로젝트는 4~5개로 충분
- 예외적인 경우만 `clamp()`나 임의값(`min-[475px]` 등)으로 보완
